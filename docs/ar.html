المشكلة

التقارير دايماً الجزء الصعب

كل مشروع بيحتاج تقارير في النهاية — لكن كتابة كويريز منفصلة لكل مصدر، لكل فورمات، لكل تينانت، مش قابل للتوسع.

😩  الطريقة القديمة تقرير واحد = كلاس كامل
⏳ MySQLRevenueReport.php+80 سطر
⏳ MongoAnalyticsReport.php+110 سطر
⏳ ExcelExportController.php+60 سطر
⏳ CsvExportController.php+45 سطر
...×12 تقرير
// متكرر في كل تقرير $query = DB::table('orders') ->join('customers', ...) ->where('tenant_id', $tenantId) ->groupBy('status') ->selectRaw('SUM(amount) as total...');
⚠ تقرير جديد = 3–5 ملفات جديدة في كل مرة
  • كل تقرير جديد بيتطلب كلاس PHP من الصفر
  • لوجيك كويري منفصل وغير متوافق بين MySQL وMongoDB
  • كود التصدير (Excel, CSV, JSON) متكرر في كل كنترولر
  • لا validation مركزي — مخاطر Injection مش ملحوظة
  • فلترة Multi-tenant بيتحطها يدوياً في كل كويري
  • ودجيتات الداشبورد ملهاش عقد binding موحد
  • الكاشينج متنسوخ بشكل مش متسق في التقارير المختلفة
الحل

بايبلاين تنفيذ من 8 خطوات

كل تقرير — بغض النظر عن المصدر — بيمر على نفس البايبلاين المحصّن قبل أي استدعاء للداتابيز.

الدخول

DSL Input

JSON object (أو PHP array) بيوصف التقرير: المصدر، الجدول، الحقول، الفلاتر، التجميعات، الترتيب، الصفحات.

1
2
النواة

DslParser

DslParser بيحوّل الـ input الخام لـ QueryDefinition مكتوبة بالنوع (immutable) مع هاش MD5 ثابت.

التحقق

QueryValidator

بيطبّق الحدود: أقصى صفوف، أقصى joins، أقصى شروط، أقصى تجميعات. بيرفض أي Operator غير مسموح.

3
4
الأمان

QuerySanitizer + ACL

QuerySanitizer بيتحقق من كل identifier بـ Regex. FieldAccessControl بيشيل الحقول المحظورة.

الكاش

Redis Cache Probe

QueryCacheManager بيشوف Redis بالهاش. مفاتيح الكاش مقسّمة بالتينانت — بيانات تينانت ميتسرّبش لتاني.

5
6
التوجيه

DataSourceResolver

بيجيب المحوّل الصح من الـ Registry: MySQLDataSource أو MongoDataSource. ممكن تضيف محوّلات Custom وقت التشغيل.

التنفيذ

المحوّل ينفّذ

MySQL بيستخدم Laravel Query Builder. MongoDB ببني بايبلاين $facet — بيانات + totalCount في رحلة واحدة للداتابيز.

7
8
النتيجة

ExecutionResult + Event

بيرجع ExecutionResult بالداتا والعدد الكلي. بيحفظ في Redis وبيطلق Event ReportExecuted.

الميزات

كل حاجة جاهزة — مفيش حاجة تربطها

🔀

DSL موحّد بـ JSON

سكيما واحدة لـ MySQL وMongoDB — حقول، فلاتر (AND/OR متداخل)، تجميعات، joins، group-by، ترتيب، صفحات.

🔌

محوّلات قابلة للتوسع

سجّل مصادر بيانات Custom وقت التشغيل. ضيف PostgreSQL، Elasticsearch، أو أي HTTP API.

⚡

Redis Query Cache

النتائج محفوظة بالهاش. مفاتيح الكاش مقسّمة بالتينانت ومبتلاش على المستوى الصح عند التحديث.

🔐

حماية من Injection

Regex Whitelist على كل اسم عمود أو جدول. القيم دايماً PDO Binding — محتش في SQL أو Mongo أبداً.

🛡️

صلاحيات على مستوى الحقول

قائمة الحقول المحظورة عالمياً + محظورة حسب الدور مع Wildcard Patterns. الحقول المحظورة تتشال تلقائياً.

📤

3 فورمات تصدير

CSV (UTF-8 BOM Streaming)، XLSX (PhpSpreadsheet)، وJSON — كلهم Streamed للعميل مباشرة بدون ملفات مؤقتة.

🏢

جاهز للـ Multi-tenant

Tenant ID بيتحط في كل كويري تلقائياً. MySQL بياخد WHERE clause، وMongoDB بياخد $match stage — صفر كود إضافي.

📊

داشبورد وودجيتات

Entities لـ Dashboards وWidgets (جدول، bar chart، line chart، KPI card، heatmap). كل ودجيت مربوط بتقرير محفوظ.

📋

سجل تنفيذ كامل

dhr_report_executions بتسجّل كل تشغيل: ميلي ثانية، صفوف، ذاكرة، cache hit، الحالة.

تجربة المطوّر

من الصفر لتقرير في دقايق

الـ Facade، وAPI HTTP، وتصدير الملفات — كلهم شغّالين من أول لحظة.

// نفّذ تقرير عبر الـ Facade — من أي مكان في التطبيق use Mostafax\ReportingEngine\Support\Facades\ReportingEngine; $result = ReportingEngine::run([ 'source' => 'mysql', 'table' => 'orders', 'fields' => [ ['column' => 'id', 'alias' => 'order_id'], ['column' => 'customer_name'], ], 'aggregations' => [ ['function' => 'sum', 'column' => 'amount', 'alias' => 'revenue'], ['function' => 'count', 'column' => 'id', 'alias' => 'total' ], ], 'filters' => [ 'operator' => 'AND', 'conditions' => [ ['field' => 'status', 'operator' => '=', 'value' => 'completed'], ], ], 'group_by' => ['status'], 'order_by' => [['column' => 'revenue', 'direction' => 'desc']], 'pagination' => ['page' => 1, 'per_page' => 25], ]); // $result هو ExecutionResult value object $result->data; // array بالصفوف $result->total; // العدد الكلي (بدون LIMIT) $result->metadata->executionTimeMs; // مثلاً 12.4 $result->metadata->cacheHit; // true لو جاي من Redis
// POST /api/reporting/run — تنفيذ ad-hoc بدون حفظ POST /api/reporting/run Content-Type: application/json { "definition": { "source": "mongodb", "table": "analytics", "aggregations": [ { "function": "count_distinct", "column": "userId", "alias": "unique" } ], "pagination": { "page": 1, "per_page": 10 } } } // POST /api/reporting — احفظ تقرير POST /api/reporting { "name": "إيرادات حسب الحالة", "definition": { ... } } // POST /api/reporting/{id}/run — نفّذ تقرير محفوظ POST /api/reporting/{id}/run { "page": 2, "per_page": 50 } // GET /api/reporting/{id}/export — Stream تحميل ملف GET /api/reporting/{id}/export?format=xlsx GET /api/reporting/{id}/export?format=csv GET /api/reporting/{id}/export?format=json
// اعمل محوّل مخصص — مثلاً PostgreSQL class PostgreSQLDataSource implements DataSourceInterface { public function supports(string $sourceType): bool { return $sourceType === 'pgsql'; } public function query(QueryDefinition $def): ExecutionResult { $rows = DB::connection('pgsql') ->table($def->table)->>...; return new ExecutionResult($rows, $total, $metadata); } } // سجّله في AppServiceProvider $this->app->make(DataSourceResolver::class) ->register('pgsql', new PostgreSQLDataSource());
// Stream تصدير مباشر من الكنترولر public function download(Request $req, string $id): StreamedResponse { // Streams مباشرة — بدون ملف مؤقت، بدون استهلاك ذاكرة return $this->export->exportById( reportId: $id, format: $req->query('format', 'csv'), // csv | xlsx | json userRoles: $req->user()->roles(), ); } // ExporterFactory قابل للتوسع — سجّل فورمات مخصصة: $factory->register('parquet', new ParquetExporter());

bash — laravel artisan
$ composer require mostafax/dynamic-hybrid-reporting-engine
✓ تم التثبيت بنجاح
 
$ php artisan vendor:publish --tag=reporting-engine-config
✓ Config published → config/reporting-engine.php
 
$ php artisan vendor:publish --tag=reporting-engine-migrations
✓ Migrations published → database/migrations/
 
$ php artisan migrate
✓ dhr_reports تم الإنشاء
✓ dhr_report_executions تم الإنشاء
✓ dhr_dashboards تم الإنشاء
✓ dhr_widgets تم الإنشاء
